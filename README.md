# ⚾ 스포츠 기사 감정 분석 시스템

> Streamlit 기반 웹 애플리케이션으로, **야구 뉴스 기사 텍스트(.txt)** 를 업로드하면  
> **감정 분석 결과**, **긍정/부정 분포**, **핵심 키워드**, **워드클라우드**를 시각화하여 보여주는 프로젝트입니다.

🔗 **배포 링크**  
[https://baseball-eakcz9yj5vkkjrhbmbxjja.streamlit.app/](https://sports-news-sentiment-analysis-bfhkkugoheypcu7kxjydbq.streamlit.app/)

---

## 📌 프로젝트 개요

일반적인 감정 분석 모델은 스포츠 기사 특유의 문맥을 충분히 반영하지 못하는 경우가 있습니다.  
예를 들어 `홈런`, `역전승`, `완봉`, `맹타` 같은 표현은 강한 긍정 의미를 가지지만,  
`실책`, `부상`, `역전패`, `무득점`, `병살타` 같은 표현은 경기 맥락에서 뚜렷한 부정 의미를 가집니다.

이 프로젝트는 단순히 모델 예측 결과만 보여주는 것이 아니라,  
**스포츠 기사 도메인에 맞는 긍정/부정 키워드 보정 로직**을 함께 적용해  
보다 자연스럽고 실용적인 감정 분석 결과를 제공하도록 구현했습니다.

또한 사용자가 복잡한 설정 없이 바로 결과를 확인할 수 있도록  
**Streamlit 기반 웹 UI**로 구성해 접근성을 높였습니다.

---

## 🚀 주요 기능

- 🔍 **감정 분석**
  - KoBERT 기반 모델을 사용해 기사 텍스트의 긍정/부정을 예측합니다.

- 📊 **감정 비율 시각화**
  - 기사별 감정 결과를 집계하여 전체 긍정/부정 분포를 그래프로 보여줍니다.

- ☁️ **워드클라우드 생성**
  - 전체 기사에서 자주 등장한 핵심 키워드를 시각적으로 확인할 수 있습니다.

- 🧠 **스포츠 기사 맞춤 보정**
  - 스포츠 기사 문맥에 특화된 긍정/부정 키워드 사전을 적용해 결과를 보정합니다.

- 🧪 **간단한 업로드 기반 분석**
  - `.txt` 파일만 업로드하면 별도 전처리 없이 바로 분석할 수 있습니다.

- 📰 **여러 기사 동시 분석**
  - 하나의 파일 안에 여러 기사를 넣고 한 번에 분석할 수 있습니다.

## 📝 사용 방법

1. 기사 내용을 `.txt` 파일로 준비합니다.
2. 여러 기사를 한 번에 분석하려면 **기사 사이를 빈 줄 5개 이상**으로 구분합니다.
3. Streamlit 화면에서 `.txt` 파일을 업로드합니다.
4. 기사별 감정 분석 결과와 점수, 핵심 키워드, 전체 감정 분포, 워드클라우드를 확인합니다.

### 입력 예시

```text
첫 번째 기사 내용




두 번째 기사 내용
```

> 기사 구분 기준: **빈 줄 5개 이상**

## 🔧 핵심 구현 내용

### 1) 감정 분석 모델 연동

`sentiment_model.py`에서 로컬 `model/` 폴더의 모델과 토크나이저를 불러와  
입력 텍스트를 긍정/부정으로 분류합니다.

- 입력 문장 토크나이징
- 모델 추론 수행
- Softmax 확률 계산
- 긍정/부정 확률 중 더 높은 값을 최종 결과로 사용

즉, 단순 규칙 기반이 아니라 **사전 학습된 감정 분류 모델 기반 추론**을 중심으로 동작합니다.

### 2) 스포츠 기사 도메인 맞춤 보정

스포츠 기사에서는 일반 뉴스와 다른 표현이 많이 등장합니다.  
예를 들어 `홈런`, `완봉`, `역전승`, `MVP`는 보통 긍정 맥락이고,  
`실책`, `부상`, `역전패`, `무득점`, `논란` 등은 부정 맥락으로 해석되는 경우가 많습니다.

이를 반영하기 위해 `app.py`에 스포츠 기사용 키워드 리스트를 구성하고,  
기사 내 키워드 포함 여부를 확인해 모델 결과를 보정하도록 구현했습니다.

#### 긍정 키워드 예시
- `승리`
- `대승`
- `역전승`
- `홈런`
- `활약`
- `맹타`
- `결승타`
- `끝내기`
- `무실점`
- `완봉`
- `연승`
- `호투`
- `MVP`

#### 부정 키워드 예시
- `패배`
- `병살타`
- `실책`
- `무득점`
- `패전`
- `부진`
- `역전패`
- `부상`
- `이탈`
- `불안`
- `논란`
- `퇴장`
- `충격`
- `사고`

이 과정을 통해 일반 감정 분석보다 **스포츠 기사 문맥에 조금 더 적합한 결과**를 제공할 수 있도록 했습니다.

### 3) 기사 분리 로직

업로드된 `.txt` 파일에서 여러 기사를 동시에 처리할 수 있도록  
**빈 줄 5개 이상을 기준으로 기사 단위 분리**를 수행합니다.

이를 통해 하나의 파일 안에 여러 기사 내용을 넣고  
한 번에 분석할 수 있도록 구성했습니다.

### 4) 키워드 추출

`keyword_extractor.py`에서는 기사 본문에서  
한글 2글자 이상 단어를 중심으로 키워드를 추출합니다.

주요 방식은 다음과 같습니다.

- 정규표현식으로 한글 단어 추출
- 불용어 제거
- 단어 빈도 계산
- 상위 키워드 반환

이를 통해 각 기사별 핵심 키워드를 보여주고,  
전체 기사 기준 키워드를 누적하여 워드클라우드 생성에도 활용했습니다.

### 5) 시각화

사용자가 결과를 직관적으로 확인할 수 있도록 다음과 같은 시각화 요소를 구현했습니다.

- 기사별 감정 결과 출력
- 긍정/부정 점수 표시
- 진행률 바 형태의 감정 강도 표시
- 전체 기사 감정 분포 막대 그래프
- 전체 키워드 워드클라우드

특히 한글 시각화를 위해 `NanumGothic.ttf` 폰트를 적용하여  
그래프와 워드클라우드에서 한글 깨짐 문제를 최소화했습니다.

## 🚧 한계 및 개선 방향

### 한계

- 현재는 **긍정 / 부정 이진 분류**만 지원합니다.
- 키워드 추출이 빈도 기반이라 문맥적 중요도를 완벽히 반영하지 못합니다.
- 선수, 팀, 감독 등 개체별 감정 분석은 지원하지 않습니다.
- 야구 기사 중심 보정 규칙이므로 일반 뉴스나 타 종목 기사에는 정확도가 떨어질 수 있습니다.
- 중립적이거나 복합적인 기사에 대해서는 해석이 단순화될 수 있습니다.

### 개선 방향

- **중립 클래스 추가**
- 선수 / 팀 / 감독 단위 **개체별 감정 분석**
- 형태소 분석 기반 키워드 추출 고도화
- 기사 자동 요약 기능 추가
- 야구 외 축구 / 농구 등 **종목별 확장**
- 날짜, 팀명, 선수명 기반 필터링 기능 추가
- 기사별 결과 저장 및 다운로드 기능 추가

## 🙌 프로젝트 의의

이 프로젝트는 단순히 감정 분류 모델을 연결하는 데서 끝나지 않고,  
**스포츠 기사라는 도메인 특성**을 반영해 결과를 보정하고  
사용자가 바로 해석할 수 있도록 시각화까지 연결한 프로젝트입니다.

즉,  
**모델 추론 → 도메인 보정 → 키워드 분석 → 시각화 → 웹 서비스화**  
흐름을 하나의 프로젝트 안에서 직접 구현했다는 점에 의미가 있습니다.
